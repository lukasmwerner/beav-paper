A portfolio does not need to be flashy to be useful.

For most students, the best portfolio is one that is:

- easy to update
- easy to read
- easy to link in applications
- clear about what you actually built

## A simple project writeup structure

For each project, try covering these sections:

1. what the project does
2. why you built it
3. the stack you used
4. your specific contributions
5. what you learned
6. what you would improve next

## Why markdown helps

Markdown keeps the barrier low.

You can open one file, update a few lines, commit the change, and your site is current again. That is much easier than avoiding updates because the CMS or layout feels heavy.

## What to add next

Start with your best two or three projects.

You can always expand later.
